# Contributing

Thank you for considering contributing to this project! We welcome contributions from everyone — whether it’s fixing a typo, reporting a bug, suggesting a feature, or submitting a full pull request.

## How to Contribute

### **Submit a Pull Request (PR)**
- Fork the repository.
- Create a new branch (e.g., `fix/typo` or `feature/inter-view`).
- Make your changes with clear, readable code and documentation.
- Add or update tests if needed.
- Commit your changes with a clear message (e.g., `fix: correct typo in README`).
- Push to your fork and open a PR to the `main` branch.
- Include a description of what your PR does and why.

##  Guidelines for Code Contributions

- Follow the project’s code style (e.g., formatting, naming conventions).
- Write clean, readable, and well-documented code.
- Include tests for new functionality or bug fixes.
- Update documentation if your change affects users or developers.
- Keep PRs small and focused on one change.
